# **客户端问题**

**错误代码（ERROR CODE）**

- 如果您还在使用官方启动器，我将建议您使用第三方启动器（如PCL2、HMCL、BakaXL等）

**用户名包含无效字符（Invalid characters in usernare）**

- 你的用户名（即游戏名、游戏id）只能包含*英文字母+数字+下划线*。


**Exception in thread "main" java.lang.UnsatisfiedLinkError:** ~C:\Users\�����~

- 取消勾选 使用unicode utf 8 提供全球语言支持选项，[具体参见](https://www.bilibili.com/read/cv17560079)



**java.lang.RuntimeException** *Minecraft game provider couldn't locate the game !*

- 导入整合包时，尝试仅使用数字和英文命名。


**Description: Unexpected error**

- 如伴随 java.lang.IllegalStateException:Already tesselating!

    - 问题出在渲染的进行，目前看来删除OptiFine（高清修复）是唯一的解决方案，可尝试使用其他光影加载器（如[iris](https://www.mcmod.cn/class/3697.html)）。~同时，如果在错误报告中见到有关任何光影的内容，建议您先暂时将其删除，再尝试复刻错误。~

**Description: Exception in server tick loop**

- 伴随出现的错误报告一般十分简短，出去系统信息后仅有少数几行，在此范围内寻找有关任何modid并检查他与其他mod的兼容性问题（可能导致无法打开游戏或实体加载后出现崩溃）另，在此Description，也有可能出现因（包括服务端核心在内的）mod版本过旧导致的错误。如该Description后包含 *java.lang.OutOfMemoryError: Java heap space* 请增加您的内存分配！ 

**Description: Mod loading error has occurred**

- 伴随出现的错误报告一般十分简短，出去系统信息后仅有少数几行，在错误报告中 *Details:* 后会说明导致此次崩溃的mod和原因（可能是错误的将客户端mod放入了服务端、或缺少前置等）。 


# **服务端问题**

**Internal Exception** *io.netty.handler.codec.DecoderException* *writerIndex(数字)FooledUnsafeLirectbytebuf(ridx: 数字,widx:数字.cap:数字)*

- 因数据包过大产生的bug，安装[XL packets](https://www.mcmod.cn/class/4559.html)可有效解决这个问题。

- ~（尚未验证）可能是使用fabric/forge等mod加载器启动的客户端连接使用原版服务端核心运行的服务器导致的，请询问服务器管理员是否使用了正确的服务端核心~

**Description: Updating screen events** 

- 如伴随 *Name and ID cannot both be blank* 

    - 可能是PCL导致的，请更新您的PCL版本，并停止使用旧版PCL，[具体操作方法~UUID 改变导致物品栏丢失的修复方法~](https://afdian.net/p/f8f8650e850e11ecad6452540025c377)。

    - 可能是部分快照的漏洞导致的

    - 可能是mojang验证服务器出现了问题

**Missing Mods**

- 字面意思，缺少mod，或mod版本不符合需求。添加或更新提到的mod即可。



**Description: Mod loading error has occurred**

- mod加载时发生错误，在 *Details:* 后会给出发生错误的mod和错误信息，尝试根据错误信息解决。



**Unknow Host**

- 服务器ip地址出现错误，可能是输入错误，抑或是服务器端口设置、端口转发、内网穿透（请检查隧道是否畅通）、DNS解析等部分出错。



**Flying is not enabled on this server**

- 服务器开启了禁止飞行（即server.properties中的allow-flight=false），请联系服务器管理员解决。



**server is closing**

- 字面意思，可能是服务器自动重启或进行了手动关闭。



**无效会话**

- 服务器开启了正版认证（即server.properties中的online-mode=true），请联系服务器管理员解决。



**Thread: Server Watchdog**

- 如伴随 *ServerHangWatchdog detected that a single server tick took 60.00 seconds (should be max 0.05)* 出现。将server.properties中的max-ticking-time=60000改为max-ticking-time=-1（一般来说是因为大型结构的生成导致服务器造成了超过60s的卡顿，超出了max-ticking-time设置的最大值，如果想避免此卡顿可以尝试使用地图预加载mod提前加载地图，如：[Chunk-Pregenerator](https://www.mcmod.cn/class/3425.html)/[Chunky](https://www.mcmod.cn/class/6239.html)/[其他](https://search.mcmod.cn/s?key=%E5%8C%BA%E5%9D%97)）如果可以的话，请尽量使用[spark](https://www.mcmod.cn/class/4073.html)等mod进行刻分析，寻找导致卡顿的原因，这远比更改max-ticking-time的值要好，因为他从根本上解决了卡顿而非增加玩家的等待时间来防止崩溃。
